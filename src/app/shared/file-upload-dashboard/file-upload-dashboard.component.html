About Overview Dashboard

This overview provides comprehensive insights into file upload patterns and system health across all monitoring locations. The dashboard aggregates data from multiple ground stations to provide system-wide visibility.

Daily Trends: Shows file uploads and record counts over time, helping identify upload patterns and system availability.

Location Distribution: Displays upload volume by receiver showing meaningful names (BLR_TS1, NPLI_GPS, FRD_TS2) instead of technical codes, making it easy to identify data contribution from each timing system.

File Type Distribution: Breakdown of file formats being processed (e.g., .dat, .csv files) to understand data source variety.

Hourly Patterns: Identifies peak upload times and helps optimize processing schedules and resource allocation.

Summary Statistics Explained:

Total Files: Complete count of all uploaded files across all locations and time periods, indicating overall data volume.

Total Records: Aggregate number of individual data records processed from all files, showing detailed measurement count.

Average Quality: System-wide data quality percentage calculated as (valid records / total records) × 100% across all uploads.

Active Locations: Number of unique ground stations currently contributing data, indicating network operational coverage.

About Daily Statistics

Daily statistics provide day-by-day breakdown of file upload activity and data quality metrics across all monitoring locations.

Files: Total number of files successfully uploaded and processed on each day.

Records: Cumulative count of data records contained within all uploaded files for the day.

Locations: Number of unique ground stations that contributed data on that specific day.

Avg Quality: Average quality score (0-100%) based on data completeness, accuracy, and format compliance.

Completeness: Percentage of expected files received compared to the theoretical maximum based on operational schedule.

About Location Statistics

Location-wise breakdown shows individual performance metrics for each ground station in the time transfer monitoring network.

Location: Meaningful receiver identifier with original code in parentheses:

  • • NPLI_GPS (GZLI2P) - NPLI GPS receiver
  • • BLR_TS1 (GZLMB1) - Bangalore Time Server 1 GPS
  • • BLR_TS1 (IRLMB1) - Bangalore Time Server 1 NAVIC
  • • FRD_TS1 (GZLMF1) - Faridabad Time Server 1 GPS
  • • AHM_TS1 (GZLMA2) - Ahmedabad Time Server 1 GPS

Satellite System: Constellation type - GPS (American), NAVIC (Indian), GPS+NAVIC (Multi-constellation).

Files/Records: Total files uploaded and data records processed from each receiver during the selected time period.

Success Rate: Percentage of files successfully processed without errors, calculated as (successful uploads / total attempts) × 100%.

Status: Current operational status: Active (recent uploads), Inactive (no recent data), or Error (processing issues detected).

About Performance Metrics

System performance analysis tracks processing efficiency and resource utilization for optimal data handling.

Processing Time: Average time to parse, validate, and store each file, measured from upload to database commit.

File Size Analysis: Distribution of upload sizes helps optimize memory allocation and processing buffers.

Lines Skipped: Invalid or malformed data lines excluded during processing, indicating data quality issues.

Performance Grades: A+ (excellent <2s), A (good 2-5s), B (average 5-10s), C (slow >10s) based on processing times.

Throughput Analysis: Chart shows files processed per location over time, revealing system capacity and bottlenecks.

About Missing Files Analysis

Gap analysis identifies missing data uploads based on expected daily file patterns from each monitoring location.

MJD (Modified Julian Day): Astronomical day numbering system used for precise date tracking in satellite timing operations.

Expected File: Anticipated filename pattern based on location code and date (e.g., GZLMB1_60123.dat).

Days Missing: Number of consecutive days without data uploads, indicating potential equipment or communication issues.

Severity Levels: Low (1-2 days), Medium (3-7 days), High (8-14 days), Critical (>14 days) based on continuity requirements.

Gap Calculation: Compares expected daily uploads against actual received files, accounting for weekend/holiday schedules.
